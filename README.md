Trabalho em Sala – Desenvolvimento de Jogo (Computação Gráfica)
👥 Formação dos Grupos

    O trabalho deverá ser realizado em grupo, com:
        Mínimo: 2 alunos e no Máximo: 4 alunos.
    ❗ Não será permitido realizar o trabalho individualmente em nenhuma hipótese.

🎯 Objetivo da Atividade

Desenvolver um jogo digital funcional, aplicando os conceitos de Computação Gráfica abordados em sala, com ênfase no uso de bibliotecas relacionadas ao OpenGL.
💻 Tecnologias Permitidas

    Linguagem recomendada: Python
    Bibliotecas aceitas:
        Pygame (com uso gráfico adequado); GLFW; PyOpenGL;
        Outras que utilizem conceitos de OpenGL.

⚠️ Atenção:
Trabalhos que não utilizarem nenhuma biblioteca ou conceito relacionado ao OpenGL terão desconto na nota.
🧩 Requisitos Obrigatórios do Jogo
🔹 1. Transformações Gráficas

O jogo deve implementar pelo menos uma transformação, como:

    Translação (movimento); Rotação; Escala.

📌 O grupo deverá explicar onde e como essa transformação foi aplicada (no código ou no manual).
🔹 2. Mecânicas de Jogo

O jogo deve obrigatoriamente conter:

    ✅ Entrada por teclado e/ou mouse
    ✅ Sistema de pontuação
    ✅ Sistema de vida, estado ou progresso
    ✅ Sistema de níveis (com aumento de dificuldade)
    ✅ Tela final (Game Over ou vitória)
    ✅ Opção de reinício do jogo

📌 Explicação obrigatória:
O grupo deve descrever:

    Como funciona a pontuação
    Como funciona o sistema de vida/estado
    Como ocorre a progressão de nível

🎮 Exemplos de Jogos (Sugestões)

Os grupos podem escolher qualquer tipo de jogo, desde que atendam aos requisitos. Alguns exemplos:

    🎯 Jogo estilo Pong
    🚀 Jogo estilo Space Invaders
    🐍 Jogo estilo Snake
    🏃 Endless runner (personagem correndo e desviando obstáculos)
    🧱 Jogo estilo Breakout (quebra-blocos)
    🔫 Jogo de tiro simples (top-down)
    🧠 Jogo de desafio/reflexo (cliques rápidos, tempo, etc.)

⚠️ Esses são apenas exemplos. A criatividade será valorizada.
🎨 Critérios de Avaliação

A nota será baseada em:

    Implementação dos requisitos obrigatórios;
    Uso correto de conceitos de computação gráfica;
    Organização e funcionamento do código;
    Criatividade do jogo (diferencial);
    Complexidade gráfica (diferencial).
